# 服务端程序性能核心指标
* ‌RPS (Requests Per Second)‌
每秒请求数，衡量系统单位时间内处理的请求量，适用于网页访问、API调用等场景‌

* ‌‌TPS (Transactions Per Second)‌
每秒事务数，指服务器完成完整事务（如支付、下单）的能力，常用于金融和电商系统‌

* ‌‌QPS (Queries Per Second)‌
每秒查询数，特指数据库或搜索引擎的查询处理能力‌

* ‌‌RT (Response Time)‌
响应时间，从请求发出到接收响应的时间，直接影响用户体验‌

* ‌‌并发用户数 (Concurrency)‌
系统同时处理的活跃用户数，与TPS、RPS密切相关‌

* ‌‌吞吐量 (Throughput)‌
单位时间内处理的数据总量，通常以KB/s或MB/s为单位‌

* ‌‌CPU利用率‌：包括用户态（%User Time）和核心态（%Privileged Time）‌

* ‌‌内存‌：如可用内存（Available Bytes）、换页数（Page Faults/sec）‌

* ‌‌I/O‌：磁盘和网络吞吐量‌
